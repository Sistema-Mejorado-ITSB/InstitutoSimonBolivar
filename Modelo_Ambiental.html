Modelo Ambiental

El Modelo Ambiental es el primer paso en el Análisis Estructurado, cuyo objetivo es definir los límites, las metas y las interacciones del sistema propuesto con el entorno. Establece "qué" debe hacer el sistema sin entrar en detalles de "cómo" lo hará.

1 Declaración de Propósitos (D.P)

Teoría: Documento conciso que establece las metas y objetivos de alto nivel del nuevo sistem Responde a la pregunta: "¿Por qué se está construyendo este sistema?" y establece el criterio de éxito del proyecto. Debe ser la visión acordada entre analistas y usuarios.

Ejemplo de contenido: *El propósito del nuevo Sistema de Gestión de Inventario es reducir el tiempo de registro de entrada y salida de materiales en un 30% e incrementar la precisión del inventario físico al 95%, facilitando la toma de decisiones sobre reposición de stock.*

2 Diagrama de Contexto (D.C)

Teoría: Representación gráfica que muestra el sistema como un único proceso central. Identifica todas las entidades externas (personas, sistemas o dispositivos) que interactúan con el sistema y los flujos de datos principales que cruzan su fronter Es fundamental para delimitar claramente el alcance.

3 Lista de Acontecimientos (L.A)

Teoría: Un listado de todos los eventos externos que el sistema debe reconocer y a los que debe responder. Estos eventos pueden ser temporales (un reporte mensual), de flujo de datos (el cliente envía un pedido) o de control (un sensor detecta un fallo). Esta lista es clave para desarrollar el modelo de comportamiento posterior.
